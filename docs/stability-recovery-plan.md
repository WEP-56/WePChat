# 网络稳定性与图片结果恢复计划

更新时间：2026-07-10

## 1. 目标

为三条网络链路增加统一的可观察与恢复能力：

1. 模型提供商 HTTP/SSE 对话；
2. 局域网 WepChat Host WebSocket 与远程 Codex turn；
3. 图片任务提交、异步等待和大图下载。

用户必须立即看到连接异常、稳定错误码、重试次数和最终状态。短暂网络波动不应直接结束远程 turn 或让已经产生的内容消失。

## 2. UI 状态

主界面顶部增加连接状态条：

- 网络不稳定或断开：红色；
- 显示稳定错误码，例如 `NET-TIMEOUT`、`REMOTE-WS-CLOSED`；
- 显示“正在重连 2/5”或下载进度；
- 恢复成功后短暂显示绿色“连接已恢复”，随后自动收起；
- 五次失败后保持红色最终错误，用户可明确知道任务没有继续等待。

消息自身继续保存最终 `error`，并包含相同错误码，确保重新打开会话后仍能看到失败原因。

## 3. 统一错误分类

| 错误码 | 含义 | 默认可重试 |
| --- | --- | --- |
| `NET-OFFLINE` | 系统离线 | 是 |
| `NET-CONNECT` | DNS、TLS、CORS 或连接失败 | 是 |
| `NET-TIMEOUT` | 请求超时 | 是 |
| `STREAM-FIRST-BYTE-TIMEOUT` | SSE 首包超时 | 是，且仅在尚未收到内容时 |
| `STREAM-IDLE-TIMEOUT` | 流式响应长时间无数据 | 否，保留已有内容 |
| `HTTP-408` / `HTTP-429` / `HTTP-5XX` | 上游临时错误 | 视请求幂等性决定 |
| `REMOTE-WS-CLOSED` | Host WebSocket 意外断开 | 是 |
| `REMOTE-RECONNECT-EXHAUSTED` | 五次重连均失败 | 否 |
| `IMAGE-SUBMIT-UNKNOWN` | 生图可能已提交，但响应未返回 | 不自动重提，避免重复扣费 |
| `IMAGE-DOWNLOAD-TIMEOUT` | 图片结果下载超时 | 是 |
| `IMAGE-TOO-LARGE` | 单图超过安全上限 | 否 |

## 4. 重试策略

- 初始请求失败后最多重试 5 次；
- 指数退避约为 `0.8s / 1.6s / 3.2s / 6.4s / 10s`，附少量随机抖动；
- 用户主动停止时立即终止退避和请求；
- 普通 SSE 仅在尚未收到任何响应字节时自动重试；收到部分内容后不重发整个请求，避免重复文本、工具调用和计费；
- 生图任务提交不会因超时而盲目重发。若提供商返回任务 ID 或明确轮询 URL，则轮询原任务；否则报告 `IMAGE-SUBMIT-UNKNOWN`；
- 图片结果 URL 下载可以安全重试 5 次，不会创建新生图任务。

## 5. 远程 Codex 续接

手机端：

- WebSocket 意外关闭后保持当前 turn Promise，不立即判死；
- 最多重连 5 次；
- 记录最后收到的 Host 事件序号；
- 重连后调用 `remote.events.replay` 补播断线期间事件；
- 使用相同 `requestKey` 重发未收到响应的请求。

Host 端：

- 按 `requestKey` 缓存进行中 Promise 和近期结果；
- 同一请求在断线重发时返回原结果，不重复创建 thread/turn；
- 缓存保持有界，避免长期增长。

这样可以恢复 turn delta、完成事件和审批事件，同时避免轻微波动重复启动 Codex turn。

## 6. 图片任务与大文件

- 图片提交等待上限由 120 秒提高到 10 分钟；
- 每个逻辑任务携带稳定 `Idempotency-Key`；
- 支持响应中的显式任务状态 URL或 `Location` 轮询；
- 图片 URL 下载单次最多等待 3 分钟，失败最多重试 5 次；
- 下载期间显示进度；
- 单张图片上限提高并明确限制为 64 MB；
- 下载完成后再转 Data URL 并写入 IndexedDB，避免不完整结果进入会话；
- 超时与下载失败分开显示，避免把“任务成功但取图失败”误报成“生成失败”。

## 7. 验收

- 三条链路使用统一错误码；
- 页面能立即显示红色异常条和重试次数；
- Remote WebSocket 五次重连、事件补播和请求去重测试通过；
- SSE 首包失败会重试，已有增量后断开不会重发；
- 生图提交不会因未知结果自动重复计费；
- 8 MB 以上图片下载、转码与保存路径可运行，64 MB 以上明确拒绝；
- 用户主动停止能中断请求与退避；
- 所有新文件低于 1,000 行并通过现有检查。

## 8. 实施结果

本计划已于 2026-07-10 完成第一版实现。

- 新增 `js/network-stability.js`，统一错误码、指数退避、五次重试、停止信号和幂等键；
- 主界面新增连接状态条，断线/重试显示红色，进度显示蓝色，恢复显示绿色；
- 普通模型 SSE 增加 45 秒首包超时和 90 秒流空闲超时；仅在未收到任何响应字节时自动重试；
- Remote WebSocket 增加心跳、五次重连、Host 事件补播、pending 请求重发和前后台恢复检查；
- Host 增加 `requestKey` 有界缓存，相同请求重发不会重复创建 Codex thread/turn；
- 生图提交等待提高到 10 分钟，响应未知时不自动重提；
- 支持显式任务状态 URL/`Location` 轮询；
- 图片结果下载增加 3 分钟单次超时、五次重试、进度、类型检查与 64 MB 上限；
- 下载或轮询失败时在消息中保存恢复信息，用户可点击“重试获取已生成图片”，不会重新提交任务；
- 消息不再重复保存大图 Data URL，只保存工作区路径，旧会话仍兼容。

自动验证结果：

- Remote 断线后使用相同 `requestKey` 重发，事件序号补播连续；
- Host 两个相同 key 的请求只调用一次 Codex；
- 初始 Host 连接失败执行 5 次重连（初次加 5 次，共 6 次连接尝试）；
- SSE 首包前失败可恢复，已有部分内容后失败不会重发；
- 异步图片任务轮询成功；
- 9 MB 图片在两次下载失败后恢复，最终 Data URL 约 12 MB；
- 生图提交响应丢失时只 POST 一次并返回 `IMAGE-SUBMIT-UNKNOWN`；
- Vue 装配、全项目语法、Host 检查和 `git diff --check` 通过。

仍需 Android/HBuilderX 实机验证后台切换、真实弱网、不同 ROM 的 WebSocket 行为，以及目标图片提供商是否支持幂等键或任务状态 URL。
