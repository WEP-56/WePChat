## `web_fetch` 七个问题（精简版）

---

**① 错误信息一锅粥**
所有失败返回同一句话：`错误：请求失败（可能是网络或跨域限制）`，LLM 无法区分 DNS 挂、超时、404、CORS 拦截还是别的什么原因，没法做针对性应对。

**② `httpstat.us/200` 也失败**
一个返回 200 的正常服务被拦截，而 `httpbin.org/redirect/3`（3次跳转）却成功，说明工具层存在某些隐形的过滤规则（域名黑名单？User-Agent 限制？），排查困难。

**③ 超时机制不透明**
不知道 timeout 是多少秒，超时失败和普通失败报错一样，LLM 无法判断是该放弃还是重试。

**④ 200 状态码也被拦的诡异情况**
`httpstat.us/200` 说明问题不局限于 4xx/5xx，可能响应体为空也会被工具层当成失败，但错误信息没体现。

**⑤ 401 错误信息太简略**
只返回 `HTTP 401`，没有请求的 URL、响应体中的错误详情，LLM 很难判断是需要认证还是权限不足。

**⑥ 错误格式不统一**
`ftp://` 报错是 `错误：仅支持 http/https 地址`，其他失败是 `错误：请求失败（可能是网络或跨域限制）`，两种模板不一致。

**⑦ 成功响应缺少元信息**
成功了也只返回内容体，没有 HTTP 状态码、响应耗时、Content-Type 等上下文，LLM 无法判断响应是否是预期的。
